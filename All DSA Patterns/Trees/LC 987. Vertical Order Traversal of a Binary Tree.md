---

#### 🗝️Key Concept

- We keep an ordered_map<Vertical, ordered_map<level, multiset< values of all nodes on that vertical and level>>>.
- This creates a map of all verticals -> inside which there is a map of nodes at each level.
- Since there can be multiple nodes at the same level, we keep a ***multiset*** , which is a set(sorted) but allows duplicates.

```arduino
mp  (map<int, map<int, multiset<int>>>)
│
├─ vertical = -1
│   └─ map<int, multiset<int>>
│       └─ level = 1 → multiset {9}
│
├─ vertical = 0
│   └─ map<int, multiset<int>>
│       ├─ level = 0 → multiset {3}
│       └─ level = 2 → multiset {15}
│
├─ vertical = 1
│   └─ map<int, multiset<int>>
│       └─ level = 1 → multiset {20}
│
└─ vertical = 2
    └─ map<int, multiset<int>>
        └─ level = 2 → multiset {7}

```

---
### 🧠Intuition/Approach

- Push root with vertical=0, level=0 into queue.
- While queue isn’t empty:
    - Pop front node, add value to `map[vertical][level]`.
    - Push left child with `vertical-1, level+1`.
    - Push right child with `vertical+1, level+1`.
- Iterate map in order of verticals and levels to fill `vector<vector<int>> res`.

---
###  💡**Note : Why BFS > DFS for Vertical Traversal**

-  **Natural order:** nodes at the same vertical & level come in order automatically
-  **Less overhead:** queue vibes > DFS nested maps & multisets
-  **Faster:** BFS avoids per-node sorting stress
- **Memory-friendly:** queue is sequential, DFS recursion is chaotic
**⏱ TC:** O(N log N) | **💾 SC:** O(N)

```cpp
  void helper(TreeNode* root, int vertical, int level, map<int, map<int, multiset<int>>>& mp){
        if(root == nullptr) return;

        mp[vertical][level].insert(root->val);
        helper(root->left, vertical - 1, level + 1, mp);
        helper(root->right, vertical + 1, level + 1, mp);
    } 
```

---
#### ⏱ **Time Complexity (TC)**

- BFS visits each node once → **O(N)**
- Insertion into map → log operations: **O(N log N)** total (map keys are sorted)
- Optional sorting of nodes at same vertical+level (if using vector) → adds at most **O(k log k)** per vertical/level, still overall ≤ O(N log N)

#### 💾 **Space Complexity (SC)**

- Queue holds at most **O(W)** nodes (maximum width of tree) → O(N) worst case
- Map stores all nodes → **O(N)**
- Recap: **O(N) space** total

---

### ✅Solution

```cpp
 void helper(queue<pair<TreeNode*, pair<int, int>>>& q, 
        map<int, map<int, multiset<int>>>& mp){
      while(!q.empty()){
        int size = q.size();
        for(int i = 0; i<size; i++){
            TreeNode* node = q.front().first; 
            int vertical = q.front().second.first;
            int level = q.front().second.second;
            q.pop();
            mp[vertical][level].insert(node->val);

           if(node->left) q.push({node->left, {vertical-1, level+1}});
           if(node->right) q.push({node->right, {vertical+1, level+1}});
        }
      }
    }

    vector<vector<int>> verticalTraversal(TreeNode* root) {
        vector<vector<int>> res;
        queue<pair<TreeNode*, pair<int, int>>> q;
        if(root==nullptr) return res;
        map<int, map<int, multiset<int>>> mp; 
        //ordered_map<vertical, ordered_map<level, {values}>>
        q.push({root, {0, 0}});
        helper(q, mp);

        for(auto x : mp){
            vector<int> levelnodes;
            for(auto ln : x.second){
                for(int value : ln.second){
                    levelnodes.push_back(value);
                }
            }
            res.push_back(levelnodes);
        }

        return res;
    }
```
