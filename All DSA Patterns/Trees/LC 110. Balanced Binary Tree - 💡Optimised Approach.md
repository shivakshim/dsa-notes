#### ⌛Brute Force 

-[[LC 110. Balanced Binary Tree - 🐍Naive Approach]]

#### 🗝️Key Concept

- Instead of checking height separately and then checking balance,  
	👉 **compute height _while_ checking balance**.  
	If a subtree is unbalanced, return **-1 immediately** to bubble up an error and stop all recursion.
	
	Basically:  
	**“Height if balanced, -1 if not.”**  
	That’s the whole vibe.

### 🧠Intuition/Approach

1. If node is null → return 0 (height of empty tree).
2. Recursively get left height → if it’s -1, return -1.
3. Recursively get right height → if it’s -1, return -1.
4. If `abs(left - right) > 1` → return -1 (this node is unbalanced).
5. Otherwise return `max(left, right) + 1` (height).
6. In `isBalanced()`, return `helper(root) != -1`.


### ⌛TC | SC

|Version|Time|Space|Idea|
|---|---|---|---|
|**Naive**|O(N²)|O(N)|height computed repeatedly|
|**Optimized**|**O(N)**|O(H)|compute height + balance in 1 go, use -1 flag|

---

### ✅Solution

```cpp
 int maxheight(TreeNode* root){
        if(root == nullptr) return 0;

        int maxleft = maxheight(root->left);
        int maxright = maxheight(root->right);

        if(maxleft == -1 || maxright == -1) return -1;
        if(abs(maxleft - maxright) > 1) return -1;

        return 1+ max(maxleft, maxright);
    }
    bool isBalanced(TreeNode* root) {
       if(maxheight(root) == -1) return false;
       else return true;
    }
```
