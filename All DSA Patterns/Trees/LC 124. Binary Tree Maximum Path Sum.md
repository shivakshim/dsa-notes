---
#### 🗝️Key Concept

- Any path is made around a root node. And for any root node, the max sum path will be the val + maxleft +  maxright.
- Because for any connected path we can take ONLY 1 branch from the left & only 1 branch from the right, so we make sure this branch is the max sum branch.
- The only way the sum of root + left + right will go down is any of the left/right path sum is -ve. Hence when we find the left & right, we keep max(0, maxsum(root->left)) and same with left.

### 🧠Intuition/Approach

1. **Recurse:** compute max path sum for left + right subtrees.
2. **Drop negatives:** treat negative child paths as `0`.
3. **Update global max:** best **through-root** path = `left + right + root->val`.
4. **Return upward:** only **one side** can be taken upward → `root->val + max(left, right)`.

### ❓Note

- We are returning the max path sum , while storing the overall max path across each root.

### ⏱ **Time Complexity (TC)**

👉 **O(N)** — single DFS visit to each node exactly once.


### 💾 **Space Complexity (SC)**

👉 **O(H)** where **H = height of the tree**  
Worst case skewed: **O(N)**  
Balanced: **O(log N)**

---

### ✅Solution

```cpp
  int maxsum(TreeNode* root, int& sum){
        if(root == nullptr) return 0;

        int leftsum = max(0, maxsum(root->left, sum));
        int rightsum = max(0, maxsum(root->right, sum));
        sum = max(sum, root->val + leftsum + rightsum);
       
        return root->val + max(leftsum,rightsum);
    }
    int maxPathSum(TreeNode* root) {
        int sum = INT_MIN;
        maxsum(root, sum);
        return sum;
    }
```
