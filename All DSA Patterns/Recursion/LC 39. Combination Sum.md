#### 🗝️Key Concept

- ##### **At each index you choose:**
1. **Take the number** (stay on same index because unlimited picks)
2. **Skip the number** (move to next index)
Classic backtracking with _“re-use” allowed on take branch_.
Also:  
→ Use candidate only if `candidate[i] <= target`  
→ When `i == n`, check if `target == 0`.

### 🧠Intuition/Approach

- **If current num ≤ target → take it and stay.**
- **Else or regardless → skip it and move.**
- **When target hits 0 → save the combo.**
- **Backtrack after every take.**

### ❓Note

- We only move ahead when e can no more take this number again.
- We always skip the element.

### ⌛**1-liner:**  
👉 _Pick if possible and stay; otherwise skip to next; record path when target hits zero._

**TC:** Exponential — **O(2^(target / smallest number))**  
**SC:** **O(target)** recursion depth + path vector.

---

### ✅Solution

```cpp
  void combinationSubsets(int i, int n, int target, vector<int>& v, 
    vector<int>& candidates, vector<vector<int>>& ans){
        if(i == n){
            if(target == 0) ans.push_back(v);
            return;
        }

        if(candidates[i] <= target){
            v.push_back(candidates[i]);
            combinationSubsets(i, n, target-candidates[i], v, candidates, ans);
            v.pop_back();
        }
        //always without
        combinationSubsets(i+1, n, target, v, candidates, ans);
    }

    vector<vector<int>> combinationSum(vector<int>& candidates, int target) {
        vector<vector<int>> res;
        int n = candidates.size();
        vector<int> v;
        combinationSubsets(0, n, target, v, candidates, res);
        return res;
    }
```
