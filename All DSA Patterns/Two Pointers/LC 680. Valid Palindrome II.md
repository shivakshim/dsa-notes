🔗[Problem link](https://leetcode.com/problems/valid-palindrome-ii/description/)
Given a string `s`, return `true` _if the_ `s` _can be palindrome after deleting **at most one** character from it_.

---
#### ⌛Brute Force 

- Using nested Loops to remove each element and check if remaining strig is palindrome.

### 🧠Intuition

- **Key Concept - [[🌿Branching]]** : Branching is basically done when we have to take >=1 of the 2 choices, and we don't know which one will work. So we need to backtrack to both choices (using a helper function) and find the one that works. 
- Here when s[i] != s[j], we have 2 choices, either take (i+1, j) or (j-1, i). But what if both are valid, then we wouldn't be sure which one will give palindrome just by looking at it, so we backtrack by calculating for both, this is Branching.

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
bool isPalindrome(string s, int i, int j){
        while(i < j){
            if(s[i] != s[j])  return false;
            i++; j--;
        }
        return true;
    }
    bool validPalindrome(string s) {
        int i = 0, j = s.length() - 1;
        while(i < j){
            if(s[i] != s[j]){
                return isPalindrome(s, i+1, j) || isPalindrome(s, i, j-1);  //branching code snippet
                }
            i++; j--;
        }
        return true;
    }
```
