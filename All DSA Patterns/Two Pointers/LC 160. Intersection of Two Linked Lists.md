🔗[Problem link](https://leetcode.com/problems/intersection-of-two-linked-lists/)
Return node of intersection of lists or nullptr(no intersection).

---
#### ⌛Brute Force 

- Use set/ find length of both lists, move the longer one first.

#### 🗝️Key Concept

- **Two-pointer redirection trick** or the **length-difference method**.
- 👉 By switching heads, both pointers travel equal total distance (A+B = B+A). If an intersection exists, they’ll meet exactly there.

### 🧠Intuition/Approach

-  Start with `p1 = headA`, `p2 = headB`.
- Move them forward one step at a time.
- When a pointer hits `nullptr`, redirect it to the _other_ list’s head.
- Eventually, they either meet at the intersection or both hit `nullptr`.

### ❓Note

- IF no intersection occurs then both pointers will eventually meet at nullptr. Since we do - if null -> move to head2.

### ⌛TC 
- Brute - O(2n)
- Optimized - O(n)

### 📦SC
- Brute - O(n)
- Optimized - O(1)

---

### ✅Solution

```cpp
 ListNode *getIntersectionNode(ListNode *headA, ListNode *headB) {
        ListNode *h1 = headA, *h2 = headB;
        //move to next or move to other list
        while(h1 != h2){
            h1 = h1 ? h1->next : headB;
            h2 = h2 ? h2->next : headA;
        }
        //returns interesection or nullptr
        return h1;
    }
```
