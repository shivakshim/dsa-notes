### 🧠Intuition

- Works on **[[🔀Two-pointer interval sweep]]
- Key Concept - check for intervals by max(start1, start2) & min(end1, end2). Move the pointer that points to pair with smaller end, because its next might overlap with the larger end of other list.
- You start with `i=0` and `j=0` (first interval of each list).
- **Compare firstList[i] with secondList[j]**.
- Find overlap if any (`start = max(start1, start2)`, `end = min(end1, end2)`).
- Then — **move the pointer whose interval ends first**.
    Why? Because that it's next interval might intersect with  the interval with larger end.

##### 🔀Algorithm
- Start at the beginning of both lists.
- At each step:
    - Compute overlap if any.
    - Move the pointer with the earlier end.
- Repeat until one list ends.


### ❓Note

- One interval in `firstList` might overlap with **multiple** in `secondList` (and vice versa).

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
 vector<vector<int>> intervalIntersection(vector<vector<int>>& firstList, vector<vector<int>>& secondList) {
        vector<vector<int>> ans;
        int i = 0, j = 0;
        while(i < firstList.size() && j < secondList.size()){
           int start = max(firstList[i][0], secondList[j][0]);
           int end = min(firstList[i][1], secondList[j][1]);
           //check for overlap
           if(start <= end){
            ans.push_back({start,end});
           }
           //move the pointer with smaller end
           if(firstList[i][1] < secondList[j][1]) i++;
           else j++;
        }
        return ans;
    }
```
