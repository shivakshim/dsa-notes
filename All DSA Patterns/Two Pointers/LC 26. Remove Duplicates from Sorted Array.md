🔗[Problem link](https://leetcode.com/problems/remove-duplicates-from-sorted-array/description/)
Sort the Unique elements to front of array, return k (no. of unique elements).

---

### 🧠Intuition

- **Brute** - Outer loop(for j) scan through all elements to compute for all elements. Inner loop(j) continues for duplicates and stops at the 1st unique(at i). Then replaces (i+1 with j), increments i.

- **Optimized** - 2 Pointers at different paces.
      -  We keep 2 pointers -  1. i = current og +1.   (i-1 has the current og)
					    2. j = next expected og. 
    - move j till arr[j] != arr[i-1], the replace replace arr[i] with arr[j] increment i.

### ❓Note

- Why Loop over j and not i?
  Here i is the slow pointer j is fast pointer. i only points to the i only increments when a unique element is found, but j scans through all elements, skips duplicates, replaces i , in turn increments i.
- In **two-pointer problems** where one pointer is “fast” (scanning) and one is “slow” (writing/marking), you **always loop over the fast pointer**.
- The fast pointer’s job is to **visit every element** to detect conditions (duplicates, sum pairs, etc.).
- The slow pointer just **tracks the target position** or maintains state, and only moves when needed.

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
int removeDuplicates(vector<int>& nums) {
        int i = 1;  //duplicate to be replaced
        int j = 1;  //next og to be found and replaced with i, current og is at (i-1)
        for(j; j<nums.size(); j++){
            if(nums[j] != nums[i-1]){   //comparing j with og NOT with duplicate
                nums[i] = nums[j];
                i++;    //points to new og
            }
        }
     return i;
    }
```
 
Brute -
```cpp
 int removeDuplicates(vector<int>& nums) {
        if (nums.empty()) return 0; // Edge case: empty vector
        
        int i = 0;  //og
        int j = 1;  //next supposed og
        
        while (j < nums.size()) {
            while (j < nums.size() && nums[i] == nums[j]) {
                j++;
            }
            // Now nums[j] != nums[i], or j == nums.size()
            if (j < nums.size()) { 
                i++;
                nums[i] = nums[j];
            }
        }
        return i + 1;
    }
```
