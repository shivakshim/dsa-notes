#### ⌛Brute Force 

- Nested loops.

#### 🗝️Key Concept

- Use a hashmap + monotonic stack.
- We can easily find the next greater element using monotonic stack , the only issue is to map the next greater to the correct element in nums1, without having to use a nested loop.
- This can be done by using a map to store the element and its index in nums1 in a map.
- Now whenever we get a next greater element for ant element in nums2 we can first check if it present in nums1 and if it is, find its index from map, and put it in that index of the res.

### 🧠Intuition/Approach

- Build a map from each element in `nums1` → its index in result.
- Init result array with `-1`.
- Traverse `nums2` left → right with a stack of indices.
- While stack not empty **and curr > nums2[top]**:
    - If popped element exists in map, set its result = curr.
- Push current index onto stack.
- Return result.

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n + m)

### 📦SC
- Brute - O(1)
- Optimized - O(n)

---

### ✅Solution

```cpp
vector<int> nextGreaterElement(vector<int>& nums1, vector<int>& nums2) {
        unordered_map<int, int> mp1;
        for(int i=0; i<nums1.size(); i++){
            mp1[nums1[i]] = i;
        }
        //use mono stack on nums2
        stack<int> st;
        vector<int> res(nums1.size(), -1);
        for(int i = 0; i<nums2.size(); i++){
            while(!st.empty() && nums2[i] > nums2[st.top()]){
                int idx = st.top(); st.pop();
                if(mp1.count(nums2[idx])) //this element is present nums1
                res[mp1[nums2[idx]]] = nums2[i];
            }
            st.push(i);
        }
        return res;
    }
```
