#### ⌛Brute Force 

- Nested loops.

#### 🗝️Key Concept

- We need the **next greater element in a circular array**.  
👉 Trick: **simulate looping around twice** (`0 → 2n-1` with `i % n`), but only **push indices in the first pass**.
- Because the normal loop will calculate for all elements till the last one, and for last on e we need to loop once more till the last element.

### 🧠Intuition/Approach

- Init `res[n] = -1`, empty stack.
- Loop `i = 0 → 2n-1`:
    - Let `curr = nums[i % n]`.
    - While stack not empty **and curr > nums[st.top()]** → pop & set `res[st.top()] = curr`.
    - If `i < n`, push `i`.
- Return `res`.

### ❓Note

- We need to take j because we cannot directly do i = i%n i, because for the next loop i need i intact.

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
 vector<int> nextGreaterElements(vector<int>& nums) {
        stack<int> st;
        int n =nums.size();
        vector<int> res(n, -1);
        for(int i=0; i < n*2; i++){
            int j = i%n;  //cant directly change i 
            while(!st.empty() && nums[j] > nums[st.top()]){
                int idx = st.top();  st.pop();
                res[idx] = nums[j];
            }
            st.push(j);
        }
        return res;
    }
```
