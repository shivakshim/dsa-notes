🔗[Problem link](https://leetcode.com/problems/132-pattern/description/)
Given an array of `n` integers `nums`, a **132 pattern** is a subsequence of three integers `nums[i]`, `nums[j]` and `nums[k]` such that `i < j < k` and `nums[i] < nums[k] < nums[j]`. Return `true` _if there is a **132 pattern** in_ `nums`_, otherwise, return_ `false`_._

---
#### ⌛Brute Force 

- Nesting.

#### 🗝️Key Concept

- **We are looking for a “middle” element nums[j]** (the “3”)
- The tricky part: nums[k] must come **after j**, but be **smaller than nums[j]** and **bigger than nums[i]** (the “1”)
- **Monotonic stack helps us track all potential nums[k] efficiently**.

### 🧠Intuition/Approach

- **Scan array from right to left** to ensure nums[k] is always after nums[j].
- **Maintain a decreasing stack** of nums[j] candidates.
- **Keep a variable `second`** to track the largest nums[k] seen so far that’s smaller than the current nums[j].
- **If nums[i] < second**, return true → 132 pattern found.
- **While stack top < nums[i]**, pop stack → update `second` to popped value.
- **Push nums[i]** onto stack as a new candidate for nums[j].
- **If loop ends**, return false → no 132 pattern exists.

### ❓Note

- When i is greater than stack top, we replace j with i, and k with j.
- But when i is smaller than j, it cannot become the new k, because k lies to the right, and we are strictly moving to the left.

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(n)

---

### ✅Solution

```cpp
 bool find132pattern(vector<int>& nums) {
        stack<int> st;
        int k = INT_MIN;

        for(int i=nums.size()-1; i >= 0; i--){
           if(nums[i] < k) return true;

            while(!st.empty() && nums[i] > nums[st.top()]){
                k = nums[st.top()];
                st.pop();
            }

            st.push(i);
        }

        return false;
    }
```
