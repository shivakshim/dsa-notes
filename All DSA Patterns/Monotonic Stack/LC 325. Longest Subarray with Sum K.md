Given an array **`arr[]`** containing integers and an integer **`k`**, your task is to find the length of the longest subarray where the sum of its elements is equal to the given value **`k`**. If there is no subarray with sum equal to **`k`**, return **`0`**.

---
#### ⌛Brute Force 

- Check every possible subarray `(i, j)` and compute its sum.
- If `sum == k`, update the max length.

#### 🗝️Key Concept

- Use **prefix sums** to convert subarray sums into differences:  
	`sum(i..j) = prefix[j] - prefix[i-1]`

### 🧠Intuition/Approach

- Maintain a running `prefixSum`.
- If `prefixSum == k`, update `maxLen`.
- If `(prefixSum - k)` exists in the map, that means a subarray from that index+1 to current `i` sums to `k`
- Store each `prefixSum` in the map **only the first time** (to ensure longest subarray).
- Keep updating `maxLen` on the fly.

### ❓Note

- Initialize the map with {0,-1}. (for when answer is the 0th idx element)
- Order matters → don’t sort prefix sums.
- Store earliest occurrence of each prefix to maximize window length.
- Handles negatives too (unlike sliding-window).

### ⌛TC

- Brute – O(n²)
- Optimized – O(n)

### 📦SC

- Brute – O(1)
- Optimized – O(n) (for the hash map)

---

### ✅Solution

```cpp
 int longestSubarray(vector<int>& arr, int k) {
        unordered_map<int, int> mp;
        mp[0] = -1;
        int presum = 0;
        int ans = INT_MIN;
        
        for(int i=0; i<arr.size(); i++){
            presum += arr[i];
            
            if(mp.count(presum - k)) 
            ans = max(ans, i - mp[presum - k]);
            
            if(!mp.count(presum))
            mp[presum] = i;
        }
        return ans == INT_MIN ? 0 : ans;
    }
```
