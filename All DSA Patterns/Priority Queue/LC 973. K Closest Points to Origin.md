🔗[Problem link](https://leetcode.com/problems/k-closest-points-to-origin/description/)
Given an array of `points` where `points[i] = [xi, yi]` represents a point on the **X-Y** plane and an integer `k`, return the `k` closest points to the origin `(0, 0)`. distance = `√(x1 - x2)2 + (y1 - y2)2`

---

#### 🗝️Key Concept

- Compare **squared distances** (no `sqrt`) and use a heap to keep the best `k`. Squared distances preserve ordering and avoid floating-point/truncation bugs.
- No need of map, directly insert { idx, distance} into the pq.

### 🧠Intuition/Approach

- Compute `d = x*x + y*y` for each point.
- Maintain a **max-heap** of size `k` keyed by `d` (push `{d, idx}`, if size>k pop).
- Heap ends with `k` closest indices → collect their points.

### ❓Note

- Never take sqrt to only compare distances, because sqrt is double, if stored int it can lead to wrong res.
- Hence simply ignore sqrt, directly compare the distance, it'll still tell which is bigger.

### ⌛TC 
- Brute - O(n log n)
- Optimized - O(n log k)

### 📦SC
- Brute - O(n)
- Optimized - O(k)

---

### ✅Solution

```cpp
vector<vector<int>> kClosest(vector<vector<int>>& points, int k) {
        //k smallest -> max heap
        auto cmp = [&](auto &a, auto &b){ return a.second < b.second; };
        priority_queue<pair<int, int>, vector<pair<int, int>>, decltype(cmp)> pq(cmp);

        for(int i=0; i<points.size(); i++){
            auto x = points[i];
            int d = pow(x[0], 2) + pow(x[1], 2);
            pq.push({i,d});
            if(pq.size() > k) pq.pop(); 
        }

        vector<vector<int>> res;
        while(!pq.empty()){
            res.push_back(points[pq.top().first]);
            pq.pop();
        }
        
        return res;
    }
```
