#### 🗝️Key Concept

- IPO / Maximize Capital using Capital-Profit Pair + Max-Heap
- At each step, pick the _most profitable project_ among those you can currently afford — keep unlocking more projects as your capital grows.

### 🧠Intuition/Approach

- Pair up each project as `(capital, profit)` and sort by capital 💰
- Use a **max-heap (pq)** to store profits of all _affordable_ projects.
- For each of the `k` rounds:
    - Push all projects with `capital <= current w` into the heap.
    - If heap’s empty → break (no project affordable).
    - Pop max profit → add to `w`.
- Return final `w`.

### ❓Note

- We break the loop when pq becomes empty, because it means that we could not find any / any more projects that could be taken.

### ⏱️ **Time Complexity:**

`O(n log n + k log n)` → sorting + heap ops

### 💾 **Space Complexity:**

`O(n)` → heap + project vector

---

### ✅Solution

```cpp
int findMaximizedCapital(int k, int w, vector<int>& profits, vector<int>& capital) {
        vector<pair<int, int>> v;
        for(int i=0; i<profits.size(); i++){
            v.push_back({capital[i], profits[i]});
        }
        sort(v.begin(), v.end());

        priority_queue<int> pq;
        int i=0; 
        while(k > 0){
            while(i < v.size() && w >= v[i].first){
                pq.push({v[i].second});
                i++;
            }
            if(pq.empty()) break;
            
            w += pq.top();  pq.pop();
            k--;
        }
        return w;
    }
```
