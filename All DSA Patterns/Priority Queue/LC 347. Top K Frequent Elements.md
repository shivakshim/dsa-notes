🔗[Problem link](https://leetcode.com/problems/top-k-frequent-elements/description/)
Given an integer array `nums` and an integer `k`, return _the_ `k` _most frequent elements_. You may return the answer in **any order**.

---
#### ⌛Brute Force 

- We just need the `k` numbers that appear the most.  
- No fancy DS — just **count**, then **sort by frequency**.

#### 🗝️Key Concept

- Map for frequencies -> pq for getting the top k.
- since we have {element, freq}, we need a pq of pair<int,int> which will be implememted by vector of pairs based on the comparator.

### 🧠Intuition/Approach

-  Count frequencies in a `unordered_map<int,int>`
- Create a **min-heap** storing `{frequency, number}` pairs.
- For each element in map:
    - Push `{freq, num}` into heap.
    - If heap size > k → pop (remove smallest freq).
- Collect remaining elements in heap → that’s your top K

### ✅Solution

```cpp
 vector<int> topKFrequent(vector<int>& nums, int k) {
        unordered_map<int, int> mp;
        for(int i=0; i<nums.size(); i++){
            mp[nums[i]]++;
        }
        //get top k using pq
        auto cmp = [&](auto &a, auto &b){ return a.second > b.second; };
        priority_queue<pair<int, int> , vector<pair<int, int>> , decltype(cmp)> pq(cmp);
        for(auto p : mp){
            pq.push({p.first, p.second});
            if(pq.size() > k) pq.pop();
        }
        vector<int> res;
        while(!pq.empty()){
            res.push_back(pq.top().first);
            pq.pop();
        }
        return res;
    }
```
