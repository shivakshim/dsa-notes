#### 🗝️Key Concept

- Max heap(to track max freq : greedy choice) + greedy
- Use a **max heap** to always pick the most frequent remaining task first, and simulate “cooldown” periods in fixed-size cycles of `(n + 1)` units.  
- Heap ensures we use high-frequency tasks ASAP, minimizing idle time.

### 🧠Intuition/Approach

- Count frequencies of all tasks (using `unordered_map` or `vector<int>`).
- Push all frequencies into a **max heap**.
- While heap not empty → simulate one cycle of `(n + 1)` units:
    - Pick up to `(n + 1)` tasks (pop, decrement freq, store in temp).
    - Push remaining (freq > 0) tasks back into heap.
    - Add time = `cycle length` if heap still not empty, else actual tasks used.
- Return total `time`.

### ⌛TC 
- Brute - O(n2)
- Optimized - **O(N log 26)** → basically **O(N)** since max 26 tasks (constant),  
  but conceptually:   `O(N log K)` where `K = distinct tasks`.

### 📦SC
- Brute - O(1)
- Optimized - O(k)

---

### ✅Solution

```cpp
  int leastInterval(vector<char>& tasks, int n) {
        priority_queue<int> pq; //max heap
        
        //0. Fill the freq in map & push it in pq
        unordered_map<char, int> mp;
        for(auto task : tasks){
            mp[task]++;
        }
        for(auto x : mp){
            pq.push(x.second);
        }

        int time = 0;
        while(!pq.empty()){
            int cycle = n+1;
            vector<int> temp;
            //1. try filling possible slots for each cycle
            while(!pq.empty() && cycle > 0){
                int curr = pq.top(); pq.pop();
                curr--;
                if(curr > 0) temp.push_back(curr);
                time++; cycle--;
            }
            //2. refill pq with remaining frequencies
            for(auto t : temp)
                pq.push(t);
            //3. to add the idle time if any
            if(cycle > 0 && !pq.empty())
            time += cycle;
        }
        return time;
    }
```
