#### 🗝️Key Concept

- 2 Heaps :
1. `maxHeap` → stores the **smaller half** of numbers (top = largest of the smaller half)
2. `minHeap` → stores the **larger half** of numbers (top = smallest of the larger half)
- We need to keep the sizes balanced : Equal for even elements, maxheap has 1 extra for odd elements.

### 🧠Intuition/Approach

- Always push into **maxHeap** first.  
    (since new number might belong to smaller half)
- Move the top of **maxheap** → **minHeap** (to maintain ordering).
- If **minHeap** grows bigger than **maxHeap**, move its top back to **maxHeap**.

### ⌛TC 
- Brute - 
- Optimized - O(log n) per insertion

### 📦SC
- Brute - O(1)
- Optimized - O(n) total

---

### ✅Solution

```cpp
 priority_queue<int> maxheap;   //smaller half, top: largest of smaller half
    priority_queue<int, vector<int> , 
        greater<int>> minheap; //bigger half, /top: smallest of larger half

    MedianFinder() {

    }
    
    void addNum(int num) {
       maxheap.push(num);
       minheap.push(maxheap.top());
       maxheap.pop();

       //resize
       if(minheap.size() > maxheap.size()){
        maxheap.push(minheap.top());
        minheap.pop();
       }
    }
    
    double findMedian() {
        if(maxheap.size() == minheap.size())
            return (maxheap.top() + minheap.top()) / 2.0;
        else return maxheap.top();
    }
```
