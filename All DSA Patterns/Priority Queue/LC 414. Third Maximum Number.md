🔗[Problem link](https://leetcode.com/problems/third-maximum-number/description/)
Find Kth greatest DISTINCT element.

---
#### ⌛Brute Force 

-  Sort -> loop -> ignore duplicates ->  return n-kth.

#### 🗝️Key Concept

- Use a set to avoid checking for duplicates and pushing them to pq.

### 🧠Intuition/Approach

1. Create `set` to track distinct elements seen.
2. For each element: if not in set, insert into heap and set.
3. If heap size > k, pop smallest.
4. After scanning, heap top = k-th largest distinct.

### ❓Note

- Works nicely when **streaming data** or **online** scenarios.


**Time Complexity:**

- `O(n log k)`

**Space Complexity:**

- `O(k + n_distinct)`

---

### ✅Solution

```cpp
 int thirdMax(vector<int>& nums) {
        unordered_set<int> s;
        priority_queue<int, vector<int>, greater<int>> pq; //minheap
        for(int i = 0; i<nums.size(); i++){
            if(!s.count(nums[i])){
                s.insert(nums[i]);
                pq.push(nums[i]);
                if(pq.size() > 3) pq.pop();
            }
        }
        if(pq.size() == 3) return pq.top();
        
        int ans = 0;
        while(!pq.empty()){
            ans = pq.top();
            pq.pop();
        }
        return ans;

    }
```
