#### 🗝️Key Concept

- Treat each number as a "node" → transformation forms a linked path. Since they have mentioned either its 1 or always an endless loop, we can detect a cycle instead of trying to get 1.

### 🧠Intuition

- Make a helper function to get sum of squares of digits.
- slow = f(slow, fast = f(f(fast))).
- Now as question states, we either get fast = 1, or fast == slow(if endless loop exists).

### ❓Note

- Loops form because sum-of-squares map is bounded → numbers shrink to a limited range.
- for any fixed integer type (32-bit, 64-bit, etc.), the number of digits is **limited**.
		 A 32-bit int max = 2,147,483,647 → **10 digits**
		 A 64-bit int max ≈ 9,223,372,036,854,775,807 → **19 digits** 
		Worst case:
		81×19=153981 \times 19 = 153981×19=1539    (since max dig square can be *9x9=81*)
  So no matter how huge the starting number is, **after one transformation, it will be ≤ ~1500.**  
  From that point on, every next number is also a sum of digits squared → it **never escapes that small “cage”.**

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(n)
- Optimized - O(1)

---

### ✅Solution

```cpp
int sumofsquare(int n){
        int sum = 0;
        while(n > 0){
          int dig = n%10;
          sum += dig*dig;
          n /= 10;
        }
        return sum;
    }
    bool isHappy(int n) {
        int slow = n, fast = n;
        while(fast != 1){
            slow = sumofsquare(slow);
            fast = sumofsquare(sumofsquare(fast));
            if(slow == fast && slow != 1) return false;
        }
     return true;
    }
```
