🔗[Problem link](https://leetcode.com/problems/linked-list-cycle-ii/)
Find Start of cycle, if there is no cycle return nullptr.

---
#### ⌛Brute Force 

- Use set, whichever repeats first is the ans.

### 🧠Intuition

- We just need to add code to the Floyd's algo to return start of cycle instead of true.
- So we write the same code, instead of true we do the following when slow == false - 
   1. Move any 1 pointer say fast to the head.
   2. Start moving fast and slow by 1, wherever they meet first is the start of cycle.

- Since fast is 2x faster than slow pointer, we have the equation  
  2(x_1 + x_2) = x_1 + x_2 + x_3 + x_2

- Since X3 is where they both met, if we place other at head, both will reach start at the same time moving by 1.

### ⌛TC 
- Brute - O(n)
- Optimized - O(n)

### 📦SC
- Brute - O(n)
- Optimized - O(1)

### ✅Solution

```cpp
ListNode *detectCycle(ListNode *head) {
        if(!head || !head->next) return nullptr;
        ListNode *slow = head;
        ListNode *fast = head;
        while(fast && fast->next){
           slow = slow->next;
           fast = fast->next->next;
           if(slow == fast){
            //start process x1==x3
            fast = head;
            while(fast != slow){
                fast = fast -> next;
                slow = slow -> next;
            }
            return fast;
           }
        }
      return nullptr;
    }
```
