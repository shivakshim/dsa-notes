Detect cycle in linked list.

---
#### ⌛Brute Force 

- **Worst -> Use nested loops : Instead of checking cycle from a given point, we check cycle till that point.
```cpp
bool hasCycle(ListNode *head) {
        if(head == nullptr || head->next == nullptr) return false;
        if(head->next == head) return true;
        ListNode *p = head;
        while(p)
        {
            ListNode *q = head;
            while(q != p){
                if(p->next == q) return true;
                q = q->next;
            }
            p = p->next;
        }
        return false;
    }
```

- **Better - > Using set.**  We keep checking if any element is present in the set, if yes return true. If not store in set go on till it reaches null.
```cpp
        unordered_set<ListNode*> s;
        ListNode *p = head;
        while(p){
            if(s.find(p) != s.end()) return true;
            else s.insert(p);
            p = p -> next;
        }
        return false;
```
 
### 🧠Intuition

- We will use **[[🐢Floyd’s Cycle Detection Algorithm]]** — a.k.a. the **Tortoise and Hare**. It’s the _optimal_ solution for detecting a cycle in a linked list.

### ⌛TC 
- Brute force → O(n²)
- Set-based → O(n) time, O(n) space
- Floyd’s → O(n) time, O(1) space (best of both worlds)
### 📦SC
- Brute - O(1)
- Set-based - O(n)
- Optimized - O(1)

---

### ✅Solution

```cpp
bool hasCycle(ListNode *head) {
    if (!head || !head->next) return false;

    ListNode *slow = head;
    ListNode *fast = head;

    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) return true; // they meet = cycle
    }
    return false;
}
```
