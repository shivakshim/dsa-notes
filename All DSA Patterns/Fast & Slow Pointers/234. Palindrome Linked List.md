🔗[Problem link](https://leetcode.com/problems/palindrome-linked-list/description/)
Return true if the LinkedList is a palindrome.

---
#### ⌛Brute Force 

- use a additional ds to store the elements then check.

#### 🗝️Key Concept

- Even length: `slow` starts second half directly.
- Odd length: skip the middle (`slow = slow->next`) → then reverse second half.

### 🧠Intuition/Approach

- Find mid of the linkedlist using [[🐢Floyd’s Cycle Detection Algorithm]].
- Even length: `slow` starts second half directly.
- Odd length: skip the middle (`slow = slow->next`) → then reverse second half.
- Check if both halves are same, while(h2)

### ❓Note

- Reversing 2nd half is easy because we have clarity on where to start and end. For first it creates complications.

### ⌛TC 
- Brute - O(n)
- Optimized - O(n)

### 📦SC
- Brute - O(n)
- Optimized - O(1)

---

### ✅Solution

```cpp
 bool isPalindrome(ListNode* head) {
        ListNode *slow = head;
        ListNode *fast = head;

        while(fast && fast->next){
            slow = slow -> next;
            fast = fast -> next -> next;
        }
        //if list is even : list2 starts at mid othewise at mid+1
        ListNode *h2;
        if(fast == nullptr)  h2 = slow;
        else h2 = slow -> next;
        //Reverse the 2nd list
        ListNode *p = h2;
        ListNode *q = nullptr;
        ListNode *r = nullptr;
        while(p){
            r = q;
            q = p;
            p = p->next;
            
            q -> next = r;
        }
        h2 = q;
        //compare list1 & list2 till h2 != nullptr
        ListNode *h1 = head;
        while(h2){
            if(h1->val != h2->val) return false;
            h1 = h1 -> next;
            h2 = h2 -> next;
        }
        return true;
    }
```
