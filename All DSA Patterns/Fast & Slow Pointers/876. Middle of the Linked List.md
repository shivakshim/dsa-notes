🔗[Problem link](https://leetcode.com/problems/middle-of-the-linked-list/description/)
Find middle of linked list.

---
#### ⌛Brute Force 

- Count no. of nodes then run loop till mid node.

#### 🗝️Key Concept

- By [[🐢Floyd’s Cycle Detection Algorithm]] we know that when fast reaches end, slow is at mid.

### 🧠Intuition

- Imagine two runners: one double-speed, one normal. When the fast runner finishes the track, the slow runner is **halfway**.
- In an odd track, that’s the exact middle. 
- In an even track, halfway lands _between two nodes_, so you decide whether to stop a bit early or a bit late → first or second middle. 
- while (fast != nullptr && fast->next != nullptr)  -- gives us the 2nd middle(in even).

### ❓Note

- To get 1st middle -
```cpp
while (fast->next != nullptr && fast->next->next != nullptr)
```

### ⌛TC 
- Brute - O(2n)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
ListNode* middleNode(ListNode* head) {
        ListNode *slow = head;
        ListNode *fast = head;

        while(fast && fast->next){
            slow = slow->next;
            fast = fast->next->next;
        }
        return slow;
    }
```
