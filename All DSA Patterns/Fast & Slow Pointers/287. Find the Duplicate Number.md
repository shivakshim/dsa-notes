🔗[Problem link]()
Find duplicate when array size = n+1, with elements [1-n].

---

#### 🗝️Key Concept

- Since one value appears twice -> The cycle exists .
- Therefore, the **start of the cycle = that repeated value**.

### 🧠Intuition/Approach

- In the give constraints -> arr.size = n+1, elements = [1-n] we can be sure that there are n+1 nodes(with 1 repeated value ), and n vertices making it a linked list with a cycle.
- Step1 :  Use [[🐢Floyd’s Cycle Detection Algorithm]] to find cycle(where slow == fast).
- Step2 : Find the start of the cycle given by the algo.

### ❓Note

- We cant simply reply on the meeting point, because that might be any element in the loop. Hence start = duplicate.
 👉 The duplicate is always the start of the cycle because **the duplicate value is the only way two different indices          can point to the same next index, which is exactly what creates the cycle in the first place.**

### ⌛TC 
- Brute - O(n)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
int findDuplicate(vector<int>& nums) {
        int slow = nums[0];
        int fast = nums[0];
        //Step1 : Cycle detection - point of meeting
       do{
            slow = nums[slow];
            fast = nums[nums[fast]];
        } while(slow != fast);
        //Step2 : find the start of cycle
        fast = nums[0];
        while(fast != slow){
            fast = nums[fast];
            slow = nums[slow];
        }
        return fast;
    }
```
