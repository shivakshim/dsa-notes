🔗[Problem link](https://leetcode.com/problems/max-sum-of-rectangle-no-larger-than-k/)
Given an `m x n` matrix `matrix` and an integer `k`, return _the max sum of a rectangle in the matrix such that its sum is no larger than_ `k`.

---
#### 🧩 Rule of thumb

- **If problem only wants the _maximum sum_** → plain Kadane works.
- **If problem wants the _maximum sum ≤ K_** (an upper bound) → plain Kadane fails → you need Kadane's compressing + prefix sums + binary search / set.

#### ⌛Brute Force 

- **2D prefix sum brute force:** O(n² m²). You _could_ build a 2D prefix sum matrix and try all rectangles using it, but:
	Checking every possible rectangle with prefix sum = **O(n²m²)** (choose top/bottom rows and left/right cols).

#### 🗝️Key Concept

- Kadane = “max subarray sum overall”. This problem = “max subarray sum ≤ k”.
- To enforce `≤ k`, we track prefix sums in a balanced BST(set) and always look for the closest prefix that doesn’t make us exceed `k`(lower bound).
- block sums can be calculated using compressing the columns for different heights. So we compress and for every compressed array we need to find the best possible(max possible sum<=k).
- To find this, we keep a prefixSet, now what we actually want is prefixSum - k should be as small as possible, (for prefix sum to be as close to k as possible).
- So we need p >= prefixSum - k : p should be greater >= difference of how much p prefixSum is bigger than k , so that after subtracting p we go prefixSum <= k, and to ensure that is it as close to k as possible, p = lower_bound of presum-k.
- then we simply find the subarray sum for this subarray and update best : best = max(best, prefixSum - *p).
   💡Note: This best is the best / max sum for this particular height.

### 🧠Intuition/Approach

-  **Fix top row**.
- **Iterate bottom row** → build a **compressed 1D array** of column sums between top..bottom.
- **Run 1D max subarray ≤ k** on compressed:
    - Maintain running prefix sum.
    - Use an ordered set to find smallest prefix ≥ (currSum − k).
    - Update best answer.
- **Track global max** across all top–bottom choices.

### ❓Note
- **Pitfall:**  
    Don’t forget to reset the compressed array for each new `top`. Otherwise sums keep stacking incorrectly.
- **Trick to remember:**  
    Think of it as **“2D Kadane + Constraint Hack”** → Kadane fails, so we patch it with prefix+set.

### ⌛TC 
- Brute - O(n6)
- Optimized - O(n3)

### 📦SC
- Brute - O(1)
- Optimized - O(n)

---

### ✅Solution

```cpp
int prefix(vector<int> &compressed, int k){
        int best = INT_MIN, presum = 0;  //returns least dif possible
        set<int> prefixset;  
        prefixset.insert(0);

        for(int i=0; i<compressed.size(); i++){
            presum += compressed[i];
            auto it = prefixset.lower_bound(presum - k);
            if(it != prefixset.end()){
                best = max(best, presum - *it);
            }
            prefixset.insert(presum);
        }
        return best;
    }
    int maxSumSubmatrix(vector<vector<int>>& matrix, int k) {
        int n= matrix.size();
        int m = matrix[0].size();
        int ans= INT_MIN;

        for(int top=0; top<n; top++){
            vector<int> compressed(m,0);
            for(int bottom = top; bottom<n; bottom++){
                for(int col=0; col<m; col++){
                    compressed[col] += matrix[bottom][col];
                    }
                ans = max(ans, prefix(compressed, k));
                }
            }
        return ans;
    }
```
