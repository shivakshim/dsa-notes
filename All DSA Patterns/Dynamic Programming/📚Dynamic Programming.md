- Memoization (Top-Down): This involves using recursion with a storage mechanism (like an array or map) to save the results of function calls ([12:35](https://www.youtube.com/watch?v=tyB0ztf0DNY&t=755s&pp=0gcJCTAAlc8ueATH)). If a subproblem has already been solved, its stored result is directly returned ([16:09](https://www.youtube.com/watch?v=tyB0ztf0DNY&t=969s)). The video uses Fibonacci numbers as an example to illustrate this ([5:01](https://www.youtube.com/watch?v=tyB0ztf0DNY&t=301s)). 
- Tabulation (Bottom-Up): This approach involves iteratively filling up a DP table starting from the base cases to compute the final result ([3:05](https://www.youtube.com/watch?v=tyB0ztf0DNY&t=185s), [23:55](https://www.youtube.com/watch?v=tyB0ztf0DNY&t=1435s)). 
- Space Optimization: This is a technique to reduce the space complexity of DP solutions, often by only keeping track of the necessary previous values instead of an entire array ([4:05](https://www.youtube.com/watch?v=tyB0ztf0DNY&t=245s), [27:05](https://www.youtube.com/watch?v=tyB0ztf0DNY&t=1625s)). For Fibonacci, this means only storing the two previous numbers ([29:01](https://www.youtube.com/watch?v=tyB0ztf0DNY&t=1741s)).
