#### 🗝️Key Concept

- You’re reversing **a sublist (between left and right)** inside a singly linked list — not the whole thing.  
	So you:  
	1️⃣ reach the node _just before_ the sublist,  
	2️⃣ reverse that portion,  
	3️⃣ reconnect the ends cleanly.

### 🧠Intuition/Approach

- **Create dummy node** → handles edge cases when reversal starts at head.
- **Move `beforeLeft`** → stop at node right _before_ `left`.
- **Mark `start` & `then`** → `start` = first node to reverse, `then` = node after `start`.
- **Iterate (right-left) times:**
    - `start->next = then->next;`
    - `then->next = beforeLeft->next;`
    - `beforeLeft->next = then;`
    - `then = start->next;`  
        _(basically insert each “then” node at the front of the sublist segment)_
- **Return `dummy->next`** as new head.

### ❓Note

- During the reversal. we keep current on the left , and q and r at nullptr, we start normally like its a new list to be reversed.

### ⏱️ **Complexity**

- **Time:** O(N) (single pass through the list)
- **Space:** O(1) (in-place reversal)

---

### ✅Solution

```cpp
ListNode* reverseBetween(ListNode* head, int left, int right) {
        if(!head || !head->next) return head;
        ListNode *dummy = new ListNode(0, head);
        ListNode *beforeleft = dummy;
        //move till before left
        for(int i=1; i<left; i++){
            beforeleft = beforeleft -> next;
        }
        //simple reversal for the given length
        ListNode *p = beforeleft->next;
        ListNode *q = nullptr, *r = nullptr;
        ListNode *tail = p; //to join later to remaining part
        for(int i=1; i <= (right - left + 1); i++){
            r = q;
            q = p;
            p = p->next;
            q -> next = r;
        }
        //connect ends
        beforeleft -> next = q;
        tail -> next = p;
  
        return dummy->next;
    }
```
