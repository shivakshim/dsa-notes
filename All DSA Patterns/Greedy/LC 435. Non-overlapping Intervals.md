🔗[Problem link](https://leetcode.com/problems/non-overlapping-intervals/description/)
Given an array of intervals `intervals` where `intervals[i] = [starti, endi]`, return _the minimum number of intervals you need to remove to make the rest of the intervals non-overlapping_.

---
#### ⌛Brute Force 

- **Concept:** Compare every pair of intervals and remove overlaps.

#### 🗝️Key Concept

- Always pick the interval that **ends earliest** → maximize space for future intervals.
- Keeping the smaller end might or might not lead to overlaps, but keeping the greater end is a "STRICTLY WORSE CHOICE".

### 🧠Intuition/Approach

- Sort intervals by **end time ascending**.
- Initialize `prevEnd = first interval end`.
- For each interval:
    - If `start < prevEnd` → overlap → remove it.
    - Else → update `prevEnd`.
- Return total removals.

### ❓Note

- Sorting happens based on the end and not the start, because we need to keep max intervals , so to remove we need the interval that has the max end.

### ⌛TC 
- Brute - O(n^2)
- Optimized - `O(n log n)` (sorting)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
int eraseOverlapIntervals(vector<vector<int>>& intervals) {
        sort(intervals.begin(), intervals.end(), [](const vector<int>& a,
         const vector<int>& b) {
         return a[1] < b[1];  // sort by end time
        });
    
        int ans = 0; int j=0;
        for(int i=1; i<intervals.size(); i++){
            if(intervals[j][1] > intervals[i][0]){  //overlap exists
                ans++;
            }
            else j = i;
        }

        return ans;
    }
```
