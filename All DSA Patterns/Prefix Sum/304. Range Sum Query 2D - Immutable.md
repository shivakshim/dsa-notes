🔗[Problem link]()
Return prefix sum for any given i,j.

---
#### ⌛Brute Force 

- nested loops.

#### 🗝️Key Concept

- In 2D prefix sum, we shift indices by `+1` (making array size `(n+1)x(m+1)`) so that the **0th row & column act as a padding of zeros**. This avoids edge-case checks (`i-1`, `j-1`) at borders.

### 🧠Intuition/Approach

- Create `presum[n+1][m+1]` initialized with 0.
- Loop `i=0→n-1`, `j=0→m-1`.
- Fill using:
  ```cpp
   presum[i+1][j+1] = mat[i][j] + presum[i][j+1] + presum[i+1][j] - presum[i][j];
   ```

- Now any submatrix `(r1,c1)→(r2,c2)` sum =
 ```cpp	
 presum[r2+1][c2+1] - presum[r1][c2+1] - presum[r2+1][c1] + presum[r1][c1];
    ```

### ❓Note
- ##### 💡 Why use `i+1, j+1`?
	Because it’s the **elegant hack** to avoid `if(i>0 && j>0)` checks. The extra row/col of 0’s = safety buffer.
	Example:  
	Matrix is `3x3`, `presum` is `4x4`.
	- `presum[0][*]` and `presum[*][0]` are all 0.
	- So you can always apply the same formula without worrying about edges.

### ⌛TC 
- Brute - O(n4)
- Optimized - O(n2)

### 📦SC
- Brute - O(1)
- Optimized - O(n*m)
