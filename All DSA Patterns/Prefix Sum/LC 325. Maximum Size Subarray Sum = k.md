🔗[Problem link](https://leetcode.com/problems/maximum-size-subarray-sum-equals-k/)
Find max length of subarray sum = k.

---
#### ⌛Brute Force 

- nested loops.

#### 🗝️Key Concept
👉 Use **prefix sums** + a **map that stores the earliest index** where each prefix sum was seen.
- If `prefix[i] - prefix[j] = k`, then subarray `(j+1 .. i)` has sum `k`.
- To maximize length, we want the **earliest `j`**.
- That’s why we only store prefix sums in the map the _first_ time we see them.

### 🧠Intuition/Approach

- Initialize `mp[0] = -1` → handles subarrays starting at index 0.
- Track `presum = 0` as you iterate.
- At each `i`, update `presum += nums[i]`.
- If `(presum - k)` exists in map → candidate subarray from `(mp[presum - k] + 1 .. i)`.
    - Length = `i - mp[presum - k]`.
    - Update `ans = max(ans, that length)`.
- If `presum` not in map, store `mp[presum] = i` (earliest occurrence).
- Return `ans`.

### ❓Note

- Length in prefix sum problems is given by right - left not roght-left+1 because 
👉At left, we don’t store “from left,” we store “ahead of left” (the prefix sum before left). That’s why no `+1` is needed later.💡

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
int maxSubArrayLen(vector<int>& nums, int k) {
    unordered_map<int,int> mp;
    mp[0] = -1;  // base case
    int presum = 0, ans = 0;
    for (int i = 0; i < nums.size(); i++) {
        presum += nums[i];
        if (mp.count(presum - k)) {
            ans = max(ans, i - mp[presum - k]);
        }
        if (!mp.count(presum)) {   // store earliest index only
            mp[presum] = i;
        }
    }
    return ans;
}
```
