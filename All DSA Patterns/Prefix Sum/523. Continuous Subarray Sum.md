🔗[Problem Link]([523. Continuous Subarray Sum](https://leetcode.com/problems/continuous-subarray-sum/))
### 🧠 Intuition (prefix sum + remainder hashmap) → **O(n)**

You don’t need to check all subarrays explicitly.  
The key idea is modular arithmetic:

1. Compute prefix sum on the fly.
2. Take `sum % k`.
    - If two prefix sums give the same remainder, it means the subarray between them is divisible by `k`.
    - Example: if `prefix[i] % k = prefix[j] % k` → then `sum(i+1 … j)` is divisible by `k`.
    - Explanation : 
        1. Prefix sum idea Let’s say:
	        prefix[i]=sum of nums[0…i]
			Now, the sum of a subarray `nums[l…r]` is:
			subarraySum(l,r) = prefix[r]−prefix[l−1]

         2. Modular arithmetic trick
			Suppose you want `subarraySum(l, r)` divisible by `k`.  
			That means  -->    
			          (prefix[r]−prefix[l−1])  %  k=0
			Which is equivalent to  -->
			          prefix[r]%k=prefix[l−1]%k

3. Use a `map<int,int>` (or `unordered_map`) to store the **earliest index** where a remainder was seen.

---
### ✍️Note - 
- Initialize map to : mp[0] = -1; // for edge case where the rem == 0.

---

## ✅Solution - 

```cpp
class Solution {
public:
    bool checkSubarraySum(vector<int>& nums, int k) {
        unordered_map<int, int> mp;
        mp[0] = -1;
        long long presum = 0; int rem = 0;
        for(int i=0; i<nums.size(); i++)
        {
            presum += nums[i];
            rem = ((presum % k) + k) % k;
            if(mp.count(rem))
            {
                if(i - mp[rem] >= 2) return true;
            }
            else mp[rem] = i;
        }
        return false;
    }
};
```
