 Perform a series of range increment operations as : Increment the values in `arr` from index **`ai`** to **`bi`** (both inclusive) by ki and find max element of result array.

---
#### 🧠Intuition - Prefix sum + Difference array
- Step 1 - Each range update can be seen as **“start adding at a, stop adding after b”**. So update range endpoint (ai, bi+1) by ki.
- Step 2 - Do prefix sum on the range update array to even out the overall values.
- In the end, we get result because for any range ai-bi : from ai prefix will keep adding up till bi, then since at bi+1 we do a -ki , it makes sum it is subtracted from total running prefix sum that involves ki from ai.

##### ❓Edge case
- bi+1 might go out of bounds -> his would mean that the right limit doesn't exists, subarray is from (ai ,n-1). So:
           if(bi+1 < n)   ans[bi+1] -= ki;
 
---

#### ✅Solution - 

```cpp
 int findMax(int n, vector<int>& a, vector<int>& b, vector<int>& k) {
        vector<int> ans(n,0);
        //1.update ranges by k
        for(int i = 0; i<a.size(); i++){
            ans[a[i]] += k[i];
            if(b[i]+1 < n) ans[b[i]+1] -= k[i];
        }
        
        //2. now take prefix sum to even out the sum
        for(int i=1; i<n ; i++){
            ans[i] = ans[i-1]+ans[i];
        }
        
        return *max_element(ans.begin(), ans.end());
    }
```
