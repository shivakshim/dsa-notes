#### 🗝️Key Concept

- **Type:** Greedy intervals / jump coverage problem
- **Goal:** Cover `[0, time]` with **minimum number of clips**
- **Core idea:** At each step, pick the clip that **starts ≤ current coverage** and **extends the coverage the farthest**
- **Sorting:**
    - Primary → `start` ascending
    - Secondary → `end` descending (if starts tie, pick clip that goes farthest)
- **Variables:
    - `curEnd` → farthest point currently covered
    - `nextEnd` → farthest point reachable with clips starting ≤ `curEnd`
    - `ans` → number of clips used

### 🧠Intuition/Approach

1. Sort clips by start (tie → end descending)
2. Initialize `curEnd = 0`, `nextEnd = 0`, `ans = 0`, `i = 0`
3. While `curEnd < time`:
    - Extend coverage: update `nextEnd = max(nextEnd, clips[i].end)` for all clips starting ≤ `curEnd`
    - If `nextEnd == curEnd` → cannot extend → return -1
    - Else → take this “jump”: `ans++`, `curEnd = nextEnd`
4. Return `ans` once `curEnd >= time`

### ✅Solution

```cpp
 int videoStitching(vector<vector<int>>& clips, int time) {
         sort(clips.begin(), clips.end(),
          [](const vector<int>& a, const vector<int>& b) {
        if(a[0] == b[0]) return a[1] > b[1];
        return a[0] < b[0];
    }); 
        int ans = 0 ;
        int maxend = -1;
        int end = 0;
        int i = 0;
        while(i < clips.size()){
            while(i < clips.size() && clips[i][0] <= end){
                maxend = max(maxend , clips[i][1]);
                i++;
            }
            if(maxend == end) return -1;
            ans++;
            end = maxend;
            if(maxend >= time) return ans;
        }
        return maxend >= time ? ans : -1;
    }
```
