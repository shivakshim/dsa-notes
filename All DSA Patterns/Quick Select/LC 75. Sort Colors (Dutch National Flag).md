🔗[Problem link](https://leetcode.com/problems/sort-colors/)

---
#### ⌛Brute Force

- Just sort the array (O(n log n)).
- Or count 0s, 1s, 2s → rewrite array.

---

#### 🗝️Key Concept

- 3-way partition in **one pass**:
    - left zone for 0s
    - middle zone for 1s
    - right zone for 2s

---

### 🧠Intuition/Approach

- Keep 3 pointers:  
    `low`, `mid`, `high`
    `0 → low-1      => 0s low → mid-1    => 1s mid → high     => unknown high+1 → end   => 2s
- Iterate till mid ≤ high.
    - If 0 → swap(mid, low) → low++, mid++
    - If 1 → mid+
    - If 2 → swap(mid, high) → high--

---

### ❓Note

- The key mental model:  
    Mid points to the _current unprocessed element_,  
    not the “1s region”.  
    That’s why mid moves after every process except when you swap with high.


---

### ⌛TC

- Brute - O(n log n)
- Optimized - O(n)

### 📦SC

- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
 void sortColors(vector<int>& nums) {
        int low = 0, mid = 0, high = nums.size() - 1;

        while (mid <= high) {
            if (nums[mid] == 0) {
                swap(nums[low++], nums[mid++]);
            } 
            else if (nums[mid] == 1) {
                mid++;
            } 
            else { // nums[mid] == 2
                swap(nums[mid], nums[high--]);
            }
        }
    }
```
