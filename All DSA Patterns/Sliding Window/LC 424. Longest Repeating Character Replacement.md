🔗[Problem link](https://leetcode.com/problems/longest-repeating-character-replacement/description/)
You are given a string `s` and an integer `k`. You can choose any character of the string and change it to any other uppercase English character. You can perform this operation at most `k` times.
Return _the length of the longest substring containing the same letter you can get after performing the above operations_.

---

#### 🗝️Key Concept

- [[💡Sliding window + track max frequency]] : for each window we check.
- if the window is valid, then we update ans , else we shrink l lazily , why? -> longest/duplicates allowed.

### 🧠Intuition/Approach

1. Use sliding window with `l`, `r`, and a freq map.
2. Update freq of `s[r]` and track `maxFreq`.
3. If `(window size – maxFreq > k)`, shrink from left.
4. Track max window length during the process.

✨ Note: lazy `maxFreq` is fine — shrinking keeps window valid.

### ❓Note

- **Key:** Overestimating `maxFreq` is safe → you may keep window larger than necessary but never miss the true max.

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(n)

---

### ✅Solution

```cpp
int characterReplacement(string s, int k) {
        int l=0, r=0, maxlen=0, maxfreq = 0;
        unordered_map<int, int> freq;
        while(r < s.length()){
            freq[s[r]]++;
            maxfreq = max(maxfreq, freq[s[r]]);

            if((r-l+1) - maxfreq > k){
                freq[s[l]]--;
                l++;
            } 

            if((r-l+1) - maxfreq <= k)
              maxlen = max(maxlen, r-l+1);

            r++;
        }
        return maxlen;
    }
```
