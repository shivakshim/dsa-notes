🔗[Problem link](https://leetcode.com/problems/sliding-window-maximum/description/)
Given an array of integers `nums`, there is a sliding window of size `k`, Return _the max sliding window_.

---
#### ⌛Brute Force 

- Nested loops.

#### 🗝️Key Concept

- `"maximum of every window"`, `"minimum in subarray"`, `"sliding window maximum"` -> use deque.
- Since we want the actual elements of each k sized window, simple sliding window cany work, we need a deqeue to get O(n).

### 🧠Intuition/Approach

- Initialize a deque.
- Iterate over the array:
    - Remove indices from front if they are **out of current window**.
    - Remove indices from back if **nums[new] > nums[back]**.
    - Push current index to back.
- If current index >= k-1 → push `nums[deque.front()]` to result.

### ❓Note

- You **don’t care about elements smaller than the current max** within the window.
- You only care about **candidates for max**, and when the window moves, drop elements that are **out of range**.

### ⌛TC 
- Brute - O(n * k)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
vector<int> maxSlidingWindow(vector<int>& nums, int k) {
    deque<int> dq;
    vector<int> res;
    for(int i=0; i<nums.size(); i++){
        // remove indices out of window
        while(!dq.empty() && dq.front() <= i - k) dq.pop_front();
        // remove smaller elements from back
        while(!dq.empty() && nums[dq.back()] < nums[i]) dq.pop_back();
        dq.push_back(i);
        // record max for this window
        if(i-k >= -1) res.push_back(nums[dq.front()]);
    }
    return res;
}
```
