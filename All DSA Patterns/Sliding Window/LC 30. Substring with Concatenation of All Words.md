🔗[Problem link](https://leetcode.com/problems/substring-with-concatenation-of-all-words/description/)
You are given a string `s` and an array of strings `words`. All the strings of `words` are of **the same length**.
Return an array of _the starting indices_ of all the concatenated substrings in `s`. You can return the answer in **any order**.

---
#### ⌛Brute Force 

- By comparing directly the maps , TC = O(n* k)

#### 🗝️Key Concept

- [[🪟 Sliding Window + Freq Maps Cheat Sheet]]
- We need to find all substrings of `s` that are **a concatenation of all words in `words`** (each used exactly as many times as given, order doesn’t matter, just internal order of chars in words does.
- Each word has the same length.
- Since we need concatenating of all words, we basically want all substrings oof fixed length and inside them there should be all the words in the words string.
- Valid substring length = `k * wordLen`.
- Use sliding window **per offset (0..wordLen-1)**, moving in word-sized jumps. Why? Because there might a case where the valid window starts from somewhere in between the word length. So we take the most outer lop as 
  👉 start =0 ; start < words[0].size; start++. 
- But why start < words[0].size(); suppose it is 3, now in each jump we move by 3 letters, so if we start from 0,1,2 then we basically jump to all possible 3 length words eventually.
- Maintain a frequency map, a needed map and a running match count.

### 🧠Intuition/Approach

- Precompute `needed` map of word → count.
- For each offset `start` in `0..wordLen-1`:
    - Init `window` map + `cnt = 0`.
    - Slide through `s` in steps of `wordLen`.
    - Add word on the right: increment count, if `window[word] ≤ needed[word]` → `cnt++`.
    - If window exceeds size, pop word on the left, update `cnt`.
    - If `cnt == k`, record index (`left`).
- Return all indices.

### ❓Note

- Remember to run the loop for the offset, otherwise we might miss possible substring.

### ⌛TC 
- Brute - O(n * k)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(n)

---

### ✅Solution

```cpp
  vector<int> findSubstring(string s, vector<string>& words) {
        unordered_map<string, int> needed;
        vector<int> ans;

        int k = words.size(), j = words[0].size(), winlen = k*j;
        //initialise needed
        for(auto x : words) needed[x]++;

        for(int start = 0; start<j; start++){
            unordered_map<string,int> window;

            int i = start, cnt = 0;

            for(; i < start + winlen && i + j <= s.length() ; i += j){
            window[s.substr(i,j)]++;
            if(needed.count(s.substr(i,j)) && 
            window[s.substr(i,j)] <= needed[s.substr(i,j)]) cnt++;
            }
        
            if(cnt == k) ans.push_back(i-winlen);
        
            for(; i + j <= s.length(); i += j){
                string left = s.substr(i-winlen, j);
                window[left]--;
                if(needed.count(left) && window[left] < needed[left]) cnt--;

                string right = s.substr(i, j);
                window[right]++;
                if(needed.count(right) && window[right] <= needed[right]) cnt++;

                if(cnt == k) ans.push_back(i-winlen+j);
            }
        }
            return ans;
    }
```
