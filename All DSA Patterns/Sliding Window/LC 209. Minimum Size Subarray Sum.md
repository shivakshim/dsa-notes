🔗[Problem link](https://leetcode.com/problems/minimum-size-subarray-sum/description/)
Given an array of positive integers `nums` and a positive integer `target`, return _the **minimal length** of a_ _subarray_ _whose sum is greater than or equal to_ `target`. If there is no such subarray, return `0` instead.

---
#### ⌛Brute Force 

- Nesting.

#### 🗝️Key Concept

- Keep moving r and taking sum, till the window doesn't become valid , sum >= k.
- Once it does, keep moving l till it doesn't become invalid, while updating the minlen for each l.
- When we do this, we move l till sum <= k because next element will anyways make sum bigger and we will have to move l.

### 🧠Intuition/Approach

- Expand `r`, keep adding to `sum`.
- While `sum >= target`, shrink from `l` and update `minlen`.
- Answer is minlen or 0 if no window exists.

### ❓Note

- Works **only** because all `nums[i] > 0`.
- With negatives → shrinking may _increase_ sum, so sliding window fails.
- Alternative approach: prefix sum + binary search (`O(n log n)`), but slower.

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
int minSubArrayLen(int target, vector<int>& nums) {
        int l=0, r=0, sum = 0, minlen = INT_MAX;
        while(r < nums.size()){
            sum += nums[r];

            while(sum >= target){
                minlen = min(minlen, r-l+1);
                sum -= nums[l++];
            }
            r++;
        }
        return minlen == INT_MAX ? 0 : minlen ;
    }
```
