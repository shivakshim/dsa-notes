🔗[Problem link](https://leetcode.com/problems/maximum-points-you-can-obtain-from-cards/description/)
Pick exactly `k` cards from **either end** of the array to maximize the total points.

---
#### ⌛Brute Force 

- nested loops.

#### 🗝️Key Concept

- Greedy 2-pointer approach **fails**; must consider all splits.
- Sliding window trick: find n-k sized minimum sum window → maximize picked points = total sum − min sum of subarray of length `n-k`.

### 🧠Intuition/Approach

- Compute total sum of all cards.
- Set window size = `n-k`.
- Slide window over array to find **min sum subarray of length n-k**.
- Return `totalSum - minSumSubarray`.

### ❓Note: Why greedy 2 pointers fail here

- **Greedy 2-pointer fails** because the locally “best” choice (bigger of left/right) may block better combinations later.
- Greedy chooses the bigger card at each step. *Example:
	`cardPoints = [1, 100, 1, 1, 1, 1, 100], k = 3`
- Greedy picks: right 100 → left 1 → right 100 = 201⚠️. Optimal: left 100 + right 100 + left 1 = 201? (depending on k)
- The optimal combination often requires **skipping a local maximum** temporarily — greedy cannot see global optimal sum.

### ⌛TC 
- Brute - O(n2)
- Optimized - O(n)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
int maxScore(vector<int>& cardPoints, int k) {
        int totalsum = 0;
        for(auto x: cardPoints) totalsum += x;

        //finding minsum window of size n-k
        int len = cardPoints.size()-k, sum = 0;
        for(int i=0; i<len; i++){
            sum += cardPoints[i];
        }
        int minsum = sum;
        for(int i=len; i<cardPoints.size(); i++){
            sum += cardPoints[i] - cardPoints[i-len];
            minsum = min(sum, minsum);
        }

        return totalsum - minsum;
    }
```
