You are given an integer array `nums` and an integer `k`. In one operation, you can choose an index of `nums` and increment the element at that index by `1`. Return _the **maximum possible frequency** of an element after performing **at most**_ `k` _operations_.

---
#### ⌛Brute Force 

- Nesting.

#### 🗝️Key Concept

- Prefix sum +  Sliding window ➡️ We basically need to find a window till r such that sum till r is [r-l+1] * nums[r].
- Core Idea - If the difference between ideal sum till r and actual sum till r is less than or equal to k, then we can add k to it and make all the elements equal to r. And count this [r-l+1].
- But if the difference between ideal sum actual sum till r is less than k then we know it can not be saves, so we simply remove the smallest element needing most k (greedily).
- For this we need to be array to be sorted.

### 🧠Intuition/Approach

- **Sort nums** → largest on right helps form target value.
- **Expand r** → add `nums[r]` into `sum`.
- **Check cost** → `(r-l+1) * nums[r] - sum`.
- **If cost > k** → shrink window: subtract `nums[l]`, `l++`.
- **Update ans** → `max(ans, r-l+1)`.

### ❓Note

- Keep the window sum and actual sum as long.

### ⌛TC 
- Brute - O(n2)
- Optimized - O(nlogn) (log n -> sorting)

### 📦SC
- Brute - O(1)
- Optimized - O(1)

---

### ✅Solution

```cpp
 int maxFrequency(vector<int>& nums, int k) {
        int l = 0, ans = INT_MIN;
        long windowsum = 0;
        sort(nums.begin(), nums.end());
        for(int r = 0; r < nums.size(); r++){
           windowsum += nums[r];
           if((long)(r-l+1)*nums[r] - windowsum > k){ //window needs more than k 
                windowsum -=nums[l]; //remove smallest element
                l++;
           }
           //window needs <= k to to be nums[r]
           ans = max(ans, r-l+1);
        }
        return ans == INT_MIN ? 1 : ans ;
    }
```
